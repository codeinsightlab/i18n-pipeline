## 核心目标
实现一个 Node.js 工程模块 i18n-pipeline，支持以下能力：

### 第一阶段（必须先完成，最小可运行版本）
1. 扫描项目中的中文字符串
   - 支持文件类型：.vue、.js、.ts、.jsx、.tsx
   - 先以正则/基础文本处理为主，优先跑通
   - 输出扫描结果（文件路径、匹配文本、行号、上下文可选）

2. 提取中文并生成 key
   - 支持基础 key 生成策略
   - 第一版允许使用 auto key，例如：
     - auto.key_001
     - auto.key_002
   - 要预留后续扩展为语义化 key 的能力
   - 相同中文文本尽量支持复用同一个 key

3. 自动生成语言包
   - 输出 zh.json
   - 格式清晰，可读性好
   - 后续便于扩展 en.json / th.json 等

4. 自动替换代码中的中文为 i18n 调用
   - Vue template 中替换为适合模板语法的形式
   - JS/TS 中替换为适合脚本语法的形式
   - 注意避免误替换注释、日志、非 UI 文本
   - 第一版宁可保守，也不要激进误替换

### 第二阶段（第一阶段跑通后再做）
5. 校验能力
   - 检测新中文硬编码
   - 检测缺失 key
   - 检测未使用 key
   - 检测多语言资源不一致

6. CLI 能力
   - 例如：
     - i18n scan
     - i18n extract
     - i18n replace
     - i18n validate

## 设计原则
